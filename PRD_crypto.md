# 币圈浮生记 — 产品需求文档 (PRD)

> 版本：v1.3.4 · 状态：已发布 · 更新日期：2026-03-31

## 一、产品概述

### 背景
基于"北京浮生记"改编的单文件 HTML5 币圈题材经营模拟游戏。
玩家扮演被老同学小王拉入 PLUS 传销骗局的普通人，欠债 $3,000（日息10%），手持 $1,000 启动资金，15天内在币圈翻盘还债。

### 核心卖点
- 黑色幽默讽刺风格
- 币圈真实事件还原（DeFi崩盘、PLUS跑路、SEC起诉、马斯克推特）
- 杠杆爆仓机制带来的高风险快感
- 新闻/情报驱动事件概率，信息即策略
- 单文件部署，无需后端

### 目标文件
`index.html` — 当前像素风实现（唯一维护版本）
`crypto-cex.html` — 旧版暗色HUD（存档备份，不再更新）

## 二、游戏核心机制

### 2.1 初始状态
| 参数 | 值 |
|------|----|
| 初始现金 | $1,000 |
| 初始负债 | $3,000 |
| 日利息 | 10%（复利） |
| 游戏天数 | 15天 |
| 初始仓位上限 | 100手 |
| 初始心态值 | 100 |

### 2.2 交易资产
| 代号 | 名称 | 价格区间 | 颜色 |
|------|------|----------|------|
| BTC | 比特币 | $8,000–126,000 | #f7931a |
| ETH | 以太坊 | $200–5,000 | #627eea |
| DOGE | 狗狗币 | $2–50 | #c2a633 |
| GPU | ETH矿机显卡 | $200–6,000 | #ff4500 |
| MINER | BTC矿机 | $200–2,000 | #0052ff |
| NFT | 总统代言 | $3,000–18,000 | #9b59b6 |
| LP | DeFi矿池 | $80–2,000 | #00c896 |
| PLUS | PLUS传销币 | $2–100 | #666666 |

### 2.3 杠杆机制
- 可选倍数：**1x / 2x**（最大2x，不开放5x/10x）
- 保证金 = 价格 ÷ 杠杆
- 爆仓价 = 入场价 × (1 - 1/杠杆)
- 2x杠杆：价格下跌50%触发爆仓，持仓清零，心态值 -15
- 同一币种只能持有一种杠杆（已持仓时锁定，清仓后可重选）

### 2.4 每日流程（doMove）
1. 天数 +1，剩余天数 -1
2. 负债 × 1.1（复利）；存款 × 1.01（DEFI利息）
3. 所有资产价格随机重置（区间内）
4. 检查爆仓（checkLiquidations）
5. 衰减 eventBoosts（各事件加成天数 -1）
6. 必然触发 1 个加权随机事件（price 70% / stress 20% / goods 5% / debt 5%）
7. 更新今日情报/新闻（写入新 eventBoosts=3）
8. 更新 UI，弹出事件弹窗

### 2.5 胜负判定
- **胜利**：心态值 > 0 且 第15天结束时 (现金 + 存款 - 负债) ≥ 0
- **失败1**：心态值归零（心态崩溃）
- **失败2**：15天到期净资产为负

## 三、UI（像素风）

### 3.1 整体风格
- 像素边框：所有面板 4px solid `#2b241d` + inset 内阴影（立体效果）
- 主面板底色：羊皮纸 `#e7d8b1`
- 电子屏：黑色背景 `#0a0f14` + 绿色荧光数字 `#6cff3d`
- 字体：`Courier New` 等宽
- 按钮：像素街机风，有按压动效

### 3.2 顶栏（Topbar）
```
[ 币圈浮生记 ]  [ 总资产USDT | 现金 | 持仓 | 存款 | 负债 ]  [ 心态 | 仓位 | 天数 ]
  蓝色渐变框      5列状态栏（总资产最宽）                     右侧状态框（红底）
```
- **总资产** = 现金 + 持仓市值 + 存款 − 负债，正数绿色，负数红色
- **右侧状态框**：默认显示心态值 / 仓位(已用/上限) / 天数；有2x杠杆爆仓风险时切换为红色警报
- 移动端（≤700px）：总资产独占一行全宽，现金/持仓/存款/负债两两分行

### 3.3 左栏（信息源）
- **信息源按钮**（4个，桌面端显示，移动端隐藏）
  - 🐦 Twitter 传闻（从 INSIDER_TIPS 随机取，触发对应 hint boost）
  - 💬 Discord 小道
  - 📊 交易所公告
  - 👤 KOL 观点
- **内幕情报卡**：每天结束自动随机更新，同步写入对应事件 boost=3
- **今日情绪屏**：电子屏显示当日情绪词（含背景色变化）

| 情绪词 | 触发条件 | 背景色 |
|--------|----------|--------|
| 按兵不动 | 无事件 | 深灰 |
| 冲！冲！冲！ | price 事件 rate > 3x | 金黄 |
| 恐慌踩踏 | price 跌幅 > 50% / debt事件 | 红色 |
| 钱包爆了 | cash 事件 | 深红 |
| 天降横财 | goods 事件 | 绿色 |

- **功能入口**（底部）
  - 🏦 DEFI存款（AAVE协议，日利率1%，独立存取滑条）
  - 💸 还款小王
  - 🧠 心理援助（$5,000 恢复心态至100）
  - 🔍 Etherscan（链上事件日志）

### 3.4 中栏（市场 + 交易）
#### 市场行情列表
列：币种图标 | 名称 | 价格 | 涨跌% | 迷你折线图（黑屏SVG）

行状态：
- 默认：羊皮纸底
- 选中：蓝色高亮
- 持仓中：左侧绿色竖边 + 图标左上角绿色 ● 徽标

#### 交易控制（2卡片网格）
**左卡：当前选择**
- 电子屏：币种名 / 当前价格 / 持仓手数 & 爆仓价 / 杠杆 / 爆仓风险%

**右卡：交易控制**
- 数量滑条（拖动实时显示手数）
- **买后剩余现金**（随滑条实时计算）/ **仓位**（已用/上限）
- 杠杆下拉框（1x / 2x；已持仓时锁定）
- [买入] 绿色 / [卖出] 红色（现金/持仓不足时自动置灰）

#### 底部操作栏
- [⏩ 下一天] 黄色（金币音效 + 按压动效）
- [全部平仓] 灰色

### 3.5 右栏（账户 + 事件 + 走势）
- **账户**：持仓市值 / 持仓数量 / 总持仓/仓位限制
- **事件流**：近3条事件卡片（图标 + 标题 + 描述）
- **走势大图**：选中币种近10天折线，2x持仓时红色虚线标注爆仓价

## 四、事件系统

### 4.1 事件类型
| 类型 | 效果 | 示例 |
|------|------|------|
| price | 某币价格 × rate | PLUS崩盘 × 0.01 |
| goods | 免费获得N手 | 空投3手LP |
| cash | 现金 × rate，心态 +hp | 钓鱼攻击损失20% |
| debt | 负债追加固定金额 | 小王追加$500 |

### 4.2 事件触发（加权随机）
- 每天**必然**触发 1 个事件
- 事件大类概率：`price 70% / stress 20% / goods 5% / debt 5%`
- 事件选取为**加权随机**：有 eventBoosts 的事件权重=6，无则=1
- 爆仓优先弹窗，之后链式弹出事件弹窗
- 事件结果写入事件流（右栏）和 Etherscan 日志

### 4.3 新闻/情报驱动概率（eventBoosts）
- `TICKER_POOL` 每条新闻和 `INSIDER_TIPS` 每条情报均有 `hint` 字段，对应某个事件 `title`
- 每天 `doMove()` 更新情报/新闻后，写入 `G.eventBoosts[hint] = 3`
- boost 在事件选取**之后**衰减（−1/天），精确生效3天
- 同一事件被多次提及取 max，不缩短剩余天数
- **概率效果**：单事件3天内至少触发一次的概率从基准 7.9% 升至约 **35%**

### 4.4 特殊事件逻辑
- **项目跑路**（PLUS）：价格降至 $0.01（bypassFloor），触发后永久锁定区间 $0.01–$0.10（`plusRugged` 标志）
- **黑客攻击**（LP）：价格可跌破底价（bypassFloor），下一天自然恢复正常区间

完整事件表见 `event.md`。

## 五、弹窗/Modal 系统

所有弹窗均为像素风（4px边框 + inset阴影），不使用系统 alert()：

| Modal | 触发 | 内容 |
|-------|------|------|
| 事件弹窗 | doMove后 | 事件图标 + 描述 |
| 爆仓弹窗 | checkLiquidations | 爆仓详情 |
| DEFI存款 | 点击功能入口 | 存入/取出独立滑条，日利率1% |
| 心理援助 | 点击功能入口 | 消费$5,000恢复心态至100 |
| 还款小王 | 点击功能入口 | 还款金额滑条 |
| Etherscan | 点击功能入口 | 全局事件日志列表 |
| 提示弹窗 | 操作错误/成功 | 像素风 warn modal |
| 游戏结束 | 胜/负判定 | 最终结算（现金/存款/欠款/净资产） |

## 七、技术约束
- 单文件 HTML5，CSS + JS 内联，无外部依赖
- 目标屏幕：桌面 1280px+（主要）/ 移动端（次要，响应式布局）
- 无后端，localStorage 本地存档（key: `cryptoGame_v2`）
- 浏览器：现代 Chrome / Safari / Edge
- 部署：push to main → GitHub Actions → Tencent COS（ap-hongkong）
